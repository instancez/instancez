Ultrabase implements the Supabase HTTP APIs so the standard `@supabase/supabase-js` client works without client-side changes for the features listed below. This page documents the current state across all three surfaces: **Auth**, **REST** (PostgREST), and **Storage**.

```js
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(ULTRABASE_URL, ULTRABASE_ANON_KEY)
```

## REST (`/rest/v1/`)

Ultrabase exposes a PostgREST-compatible HTTP surface. The supabase-js `.from()`, `.rpc()`, and query builder methods work without changes.

### At a glance

| Area | Status | Notes |
|---|---|---|
| **CRUD** ‚Äî `GET`/`POST`/`PATCH`/`DELETE`, bulk insert + `columns=`, upsert (`on_conflict=`), `?select=` on mutations | ‚úÖ Full | |
| **`Prefer` headers** ‚Äî `return=representation`/`minimal`/`headers-only`, `tx=rollback`, `count=exact`/`planned`/`estimated`, `max-affected=N`, `handling=strict` | ‚úÖ Full | |
| **Filter operators** ‚Äî `eq`/`neq`/`gt[e]`/`lt[e]`, `like`/`ilike` (incl. `all`/`any` variants), `is`, `in`, `cs`/`cd`/`ov`, `match`/`imatch`, `isdistinct` | ‚úÖ Full | |
| **Range operators** ‚Äî `sl`, `sr`, `nxl`, `nxr`, `adj` | ‚úÖ Full | |
| **Logical trees** ‚Äî `and=`, `or=`, `not.` prefixes, nested | ‚úÖ Full | |
| **Resource embedding** ‚Äî belongs-to, has-many, `!inner`, nested scope, spread (`...`), FK-hint disambiguation | ‚úÖ Full | |
| **Content types** ‚Äî `application/json`, `application/vnd.pgrst.object+json`, `text/csv` | ‚úÖ Full | |
| **JSONB path access** ‚Äî `metadata->>key` in filters and select | ‚úÖ Full | |
| **Full-text search** ‚Äî `fts`, `plfts`, `phfts`, `wfts` | ‚úÖ Full | |
| **RPC** ‚Äî `.rpc()` with scalar/SETOF/void returns, chaining, `params=single-object` | ‚úÖ Full | |
| **Aggregates** ‚Äî `count()`, `avg()`, `sum()`, `min()`, `max()`, aliases, GROUP BY, HAVING | ‚úÖ Full | |
| **Error envelope** ‚Äî `{code, message, details, hint}` with SQLSTATE + `PGRST*` codes | ‚úÖ Full | |
| **EXPLAIN** ‚Äî `Accept: application/vnd.pgrst.plan+json` returns query plan | ‚úÖ | JSON and text formats |
| **GeoJSON** ‚Äî `Accept: application/geo+json` wraps geometry columns as FeatureCollection | ‚úÖ | Requires PostGIS + geometry/geography column |
| **Multi-schema** ‚Äî `Accept-Profile`/`Content-Profile` headers | ‚úÖ | Tables can declare `schema:` in YAML; `SET LOCAL search_path` per request |
| **DB timeout** ‚Äî `Prefer: statement-timeout=N` (ms) | ‚úÖ | Capped at configured `server.timeouts.db_query` |
| **Request cancellation** (`abortSignal`) | ‚úÖ | Client-side only; HTTP context propagates to pgx |
| Views as endpoints | ‚ùå Missing | |
| Computed columns from `function(row)` | ‚ùå Missing | |

### Error codes

Constraint violations forward real SQLSTATEs:

| SQLSTATE | HTTP | Kind |
|---|---|---|
| `23505` | 409 | unique_violation |
| `23503` | 422 | foreign_key_violation |
| `23502` | 422 | not_null_violation |
| `23514` | 422 | check_violation |
| `22P02` | 422 | invalid_text_representation |
| `42501` | 403 | insufficient_privilege / RLS denial |
| `42P01` | 404 | undefined_table |
| `42703` | 404 | undefined_column |

API-level errors use `PGRST*` codes: `PGRST100` (parse), `PGRST106` (schema), `PGRST116` (single-row), `PGRST122` (strict), `PGRST124` (max-affected), `PGRST202` (unknown function), `PGRST301` (auth).

## Storage (`/storage/v1/`)

Ultrabase implements the Supabase Storage API so `supabase.storage` methods work out of the box.

| Method | Status | Notes |

| `storage.listBuckets()` | ‚úÖ | |
| `storage.getBucket(id)` | ‚úÖ | |
| `storage.from(b).list(folder)` | ‚úÖ | |
| `storage.from(b).listV2(folder)` | ‚úÖ | Cursor-based pagination, `with_delimiter` for folders |
| `storage.from(b).info(path)` | ‚úÖ | |

| `storage.from(b).update(path, file)` | ‚úÖ | Proxied, capped at bucket `max_size` |
| `storage.from(b).download(path)` | ‚úÖ | Supports image transforms via query params |
| `storage.from(b).uploadToSignedUrl(path, token, file)` | ‚úÖ | Proxied, capped at bucket `max_size` |
| `storage.from(b).emptyBucket()` | ‚úÖ | |
| Image transforms | ‚úÖ | `?width=&height=&resize=&quality=&format=` on download |
| `storage.createBucket()` | ‚ùå | Buckets are defined in YAML |
| `storage.updateBucket()` | ‚ùå | |
| `storage.deleteBucket()` | ‚ùå | |
| Resumable uploads (TUS) | ‚ùå | |
| Vector storage | ‚ùå | |

<Callout variant="warning" title="Serverless payload limits">
  The 4 proxied methods (upload, update, download, uploadToSignedUrl) stream file bytes through the server. In serverless environments, these are constrained by request/response payload limits (e.g., 6MB on AWS Lambda). For large files, use the serverless-friendly presigned URL endpoints described on the [Storage](/docs/storage) page.
</Callout>

## Testing compatibility

The integration suite runs both `postgrest-go` and `@supabase/supabase-js` against a live Ultrabase server with a real Postgres instance:

- **postgrest-go conformance** ‚Äî `internal/adapter/http/pgrupstream/conformance_test.go`: filters, transforms, embedding, mutations, error envelope
- **supabase-js harness** ‚Äî `test/integration/supabase-js/run.mjs`: auth flows (signup, signin, refresh, MFA, anonymous, OTP, PKCE, ID token, identity linking, admin ops), REST operations (CRUD, filters, embeds, upsert, RPC, explain), and storage operations (all endpoints + image transforms + serverless presigned URLs)

```sh
go test -tags=integration ./internal/adapter/http/...
```
